## 二、技术选型

| 层级 | 技术选型 | 选型理由 |
|------|----------|----------|
| **客户端引擎** | Unity 2022 LTS / Unreal 5 | 跨平台支持好，音频系统成熟，AR/VR生态完善 |
| **后端框架** | Go (Gin) / Node.js (NestJS) | 高性能，协程支持，实时性好 |
| **API 网关** | Kong / Nginx | 统一入口，限流，认证 |
| **主数据库** | PostgreSQL | 可靠的事务，JSON支持，复杂查询 |
| **缓存层** | Redis | 会话缓存，配置缓存，低延迟 |
| **对象存储** | MinIO / AWS S3 | 音频文件、视频片段静态存储 |
| **分析数据库** | ClickHouse | 时序数据聚合分析 |
| **消息队列** | Kafka / RabbitMQ | 事件驱动，解耦服务 |
| **容器化** | Docker + Kubernetes | 微服务部署，弹性伸缩 |

## 二、核心表结构设计

### 2.1 用户表 (users)

```sql
CREATE TABLE users (
    id BIGSERIAL PRIMARY KEY,
    uuid UUID NOT NULL UNIQUE DEFAULT gen_random_uuid(),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    -- 用户偏好设置 (JSONB)
    settings JSONB DEFAULT '{
        "audio_volume": 0.8,
        "music_enabled": true,
        "voice_guidance": true,
        "language": "zh-CN",
        "theme": "default"
    }'::JSONB,
    
    -- 匿名化标识 (用于数据关联，不含个人身份信息)
    anonymous_id UUID,
    
    INDEX idx_users_uuid (uuid),
    INDEX idx_users_anonymous_id (anonymous_id)
);

COMMENT ON TABLE users IS '用户主表，仅存储匿名化标识和个人偏好';
COMMENT ON COLUMN users.settings IS '用户偏好设置，不包含任何敏感个人信息';
```

### 2.2 会话表 (sessions)

```sql
CREATE TABLE sessions (
    id BIGSERIAL PRIMARY KEY,
    uuid UUID NOT NULL DEFAULT gen_random_uuid(),
    user_id BIGINT REFERENCES users(id),
    
    -- 会话时间
    started_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP WITH TIME ZONE,
    duration_seconds INTEGER,
    
    -- 体验前后的情绪标记 (1-5 评分，不记录具体情绪词汇)
    mood_before INTEGER CHECK (mood_before BETWEEN 1 AND 5),
    mood_after INTEGER CHECK (mood_after BETWEEN 1 AND 5),
    
    -- 体验完成状态
    completion_status VARCHAR(20) DEFAULT 'in_progress' 
        CHECK (completion_status IN ('in_progress', 'completed', 'abandoned')),
    
    -- 整体评分 (1-5)
    overall_rating INTEGER CHECK (overall_rating BETWEEN 1 AND 5),
    
    -- 技术环境
    device_type VARCHAR(50),
    platform VARCHAR(20),
    
    -- 创建时间
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_sessions_user_id (user_id),
    INDEX idx_sessions_uuid (uuid),
    INDEX idx_sessions_started_at (started_at)
);
```

```sql
LANGUAGE plpgsql
AS $$
BEGIN
    -- 将超过 90 天的事件移动到归档表
    EXECUTE format(
        'CREATE TABLE IF NOT EXISTS events_archive_%s PARTITION OF events_archive FOR VALUES FROM (%L) TO (%L)',

        (CURRENT_DATE - INTERVAL '90 days')::date,

    EXECUTE format(

        to_char(CURRENT_DATE - INTERVAL '90 days', 'YYYY_MM')
    );
```

# 第四部分：数据安全与隐私

## 一、隐私设计原则

```sql
-- 匿名化处理示例
CREATE OR REPLACE FUNCTION anonymize_user_data(user_id BIGINT)
RETURNS void
LANGUAGE plpgsql
AS $$
BEGIN
    -- 30 天后执行匿名化
    UPDATE users 
    SET 
        settings = settings - 'device_info' - 'location',
        anonymous_id = uuid_generate_v4()
    WHERE id = user_id 
    AND created_at < CURRENT_TIMESTAMP - INTERVAL '30 days';
END;
$$;

-- 批量匿名化存储过程
CREATE OR REPLACE PROCEDURE batch_anonymize()
LANGUAGE plpgsql
AS $$
DECLARE
    batch_size INTEGER := 1000;
    deleted_count INTEGER;
BEGIN
    LOOP
        WITH to_anonymize AS (
            SELECT id FROM users 
            WHERE created_at < CURRENT_TIMESTAMP - INTERVAL '30 days'
            AND anonymous_id IS NULL
            LIMIT batch_size
        )
        DELETE FROM to_anonymize;
        
        GET DIAGNOSTICS deleted_count = ROW_COUNT;
        
        EXIT WHEN deleted_count < batch_size;
        
        COMMIT;
    END LOOP;
END;
$$;
```
