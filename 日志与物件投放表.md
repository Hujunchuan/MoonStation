# MoonStation / Lunar Week One
## 日志与物件投放表

用途：把文字内容和空间叙事绑定到具体位置，供关卡、美术、交互与 UI 使用。

---

## 1. 投放原则

- 同一空间只放一个真正重要的故事物件，避免“信息展板化”。
- 正式工单与私人日志必须混放，形成“制度语言 / 人的语言”对照。
- 前任痕迹应当克制，不做解谜游戏式高亮。
- 能被摸到的东西优先于只能被读到的东西。

## 3. 终端与日志投放

### 3.1 Terminal A - Airlock Log

- 位置：Airlock Spine 右侧壁面。
- 首次可读：Day 1。
- 呈现方式：值守交接终端。
- 推荐内容：
  - `COM_GROUND_D1_01`
  - `D1_COM_EARTH_01`
- 作用：让玩家理解“这是排班值守，不是英雄降临”。

### 3.2 Terminal B - Quiet Deck Maintenance Note

- 位置：Quiet Deck 长凳侧边小终端。
- 首次可读：Day 2。
- 呈现方式：半损坏维护页，带一条私人备注。
- 推荐内容：
  - `LOG_QY_D2_01`
  - `LOG_QY_D2_02`
- 作用：把 Quiet Deck 从“漂亮空间”转成“被人真正用来活下去的空间”。

### 3.3 Terminal C - Systems Ring Service Console

- 位置：Systems Ring 主维护台。
- 首次可读：Day 3。
- 呈现方式：标准工单里夹着四拍记号。
- 推荐内容：
  - `LOG_QY_D3_01`
  - `LOG_QY_D3_02`
- 作用：让“节律”第一次进入维护语言。

### 3.4 Terminal D - Anomaly Overlay

- 位置：Day 4 异常时临时覆盖在主维护台。
- 首次可读：Day 4。
- 呈现方式：异常警告下插入私人句子。
- 推荐内容：
  - `LOG_QY_D4_01`
  - `LOG_QY_D4_02`
- 作用：盟友攻击，直接切进主角的心理问题。

### 3.5 Terminal E - Design Archive

- 位置：Quiet Deck 背后检修面板或独立小终端。
- 首次可读：Day 5。
- 呈现方式：人因设计档案节选。
- 推荐内容：
  - `D5_COM_EARTH_01`
  - 研究备注文本（见 `旁白_日志_通信文本库.md`）
- 作用：让玩家理解 Ritual Valve 与 Quiet Deck 是制度性设计，而非个人玄学。

### 3.6 Terminal F - Final Report Screen

- 位置：Window Gallery 回身可见的报告终端。
- 首次可读：Day 7。
- 呈现方式：结论栏待填写。
- 推荐内容：
  - `D7_COM_EARTH_02`
  - `D7_VO_OPEN_02`
- 作用：让最终道德决定从抽象价值观落到一个真实输入框。

## 4. 物件叙事投放建议

### 4.1 Quiet Deck

- 必须有的东西：
  - 金属杯
  - 刻字长凳
  - 小终端
- 不该有的东西：
  - 太多装饰品
  - 明亮鼓励式海报
  - 明显的“疗愈空间”符号

### 4.2 Systems Ring

- 必须有的东西：
  - 被频繁触碰的高亮磨损区
  - 胶带标签
  - 工单终端
- 不该有的东西：
  - 过于整洁像样板房
  - 每个面板都一模一样没有使用痕迹

### 4.3 Ritual Valve

- 必须有的东西：
  - 手套摩擦磨损
  - 单独冷光
  - 周围比其他空间更少的功能提示
- 不该有的东西：
  - 太神秘、太宗教化的造型
  - 过强的特效光柱

### 4.4 Window Gallery

- 必须有的东西：
  - 靠近玻璃后的冷凝痕迹
  - 地球升起时的尺度感
  - 回头就能看见报告终端
- 不该有的东西：
  - 胜利纪念碑式构图
  - 情绪煽动式音乐提示

## 5. 与文本库的直接对表关系

建议建立一个简单映射表：

- `LOG_QY_D2_01` -> Terminal B
- `LOG_QY_D3_02` -> Terminal C / Quiet Deck 刻字意象
- `LOG_QY_D4_01` -> Terminal D
- `D5_RIT_01 ~ D5_RIT_05` -> Ritual Valve / Deep Ritual Sequence
- `D7_VO_OPEN_02` -> Report Terminal 交互后触发
- `D7_VO_WORK_01` -> Window Gallery 首次凝视触发

---

## 6. 实现顺序建议

如果按制作优先级排：

1. 先做 Terminal B、Terminal C、Terminal D。
2. 再做 Quiet Deck 长凳刻字与 Ritual Valve 磨损。
3. 最后做 Report Terminal 与 Window Gallery 的联动。
