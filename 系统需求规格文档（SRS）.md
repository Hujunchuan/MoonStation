## 一、系统概述

### 1.1 产品定位

Lunar Week One是一个**沉浸式心理-文明模拟体验产品**，介于实验性游戏、体验式纪录片与心理仪式系统之间。产品核心目标是验证：人在极端环境（孤立、低刺激、高不确定）中，体验是否真的能够改变心理结构——提升心智稳定性、增强当下感、促进意义建构。

**这不是一个冥想App，也不是娱乐向游戏**，而是通过模拟未来文明生活方式，让用户在30分钟内经历一次"心理结构重塑"的实验性体验。

### 1.2 核心设计原则

| 原则    | 说明                   |
| ----- | -------------------- |
| 无失败惩罚 | 用户行为不触发"失败"状态，仅有状态变化 |
| 无竞争   | 完全排除对抗、排名、比较元素       |
| 节奏极慢  | 所有交互响应时间延长30%-50%    |
| 克制表达  | 避免情绪煽动、英雄叙事、成就反馈     |
| 去娱乐化  | 强调"存在"而非"有趣"         |

### 1.3 目标用户画像

| 维度   | 描述                            |
| ---- | ----------------------------- |
| 核心用户 | 25-45岁，对正念、冥想有兴趣但排斥宗教化表达的城市白领 |
| 次要用户 | 对太空文明、科幻生活方式有兴趣的技术爱好者         |
| 潜在用户 | 心理咨询师、疗愈行业从业者（作为工具使用）         |

### 1.4 MVP范围定义

**时间边界**：体验时长固定30分钟，不可延长或缩短
**平台边界**：PC桌面端首发，VR版本在MVP后考虑
**内容边界**：7个月球日单元，每单元约4分钟，共28分钟+2分钟缓冲

## 三、功能需求规格

### 3.1 模块一：模拟建筑与资源管理系统

**模块目标**：通过极简的资源管理交互，建立用户的"文明秩序感"，抵消虚无、焦虑、失控感。

**设计原则**：
- 无失败惩罚，资源指标仅在安全区间内波动
- 交互响应时间比常规游戏慢40%
- 所有反馈为视觉+音频双重编码

#### 3.1.1 能源管理子系统

**功能描述**：用户通过调整太阳能板角度来维持基地能源供应。

| 需求编号 | 功能描述 | 用户故事 | 交互规格 |
|----------|----------|----------|----------|
| PM-ENG-01 | 太阳能板可视化 | 用户可以看到一个极简风格的太阳能板模型 | 3D模型，面板有柔和的光泽材质 |
| PM-ENG-02 | 角度调整交互 | 用户点击太阳能板进行角度微调 | 点击→旋转动画（2秒完成）→能量输出变化 |
| PM-ENG-03 | 能源状态反馈 | 用户知道当前能源是否充足 | 基地灯光亮度变化（非HUD），低能源时有低频嗡鸣渐强 |
| PM-ENG-04 | 自动恢复机制 | 能源不会耗尽 | 角度偏差时，能源缓慢下降但不触发警告 |

#### 3.1.2 氧循环监控子系统

**功能描述**：用户观察氧循环系统的运作状态，无需主动干预。

| 需求编号 | 功能描述 | 用户故事 | 交互规格 |
|----------|----------|----------|----------|
| PM-OXY-01 | 氧循环可视化 | 用户看到一个抽象化的氧循环图 | 流动的光粒子，从进气口到出气口 |
| PM-OXY-02 | 氧气浓度指示 | 用户感知氧气浓度状态 | 基地内光线色调变化（暖→冷） |
| PM-OXY-03 | 异常事件（Day 4） | 用户遇到氧循环波动 | 粒子流动加速/减速，光线闪烁，无警报声 |

#### 3.1.3 水回收管理子系统

**功能描述**：监控水回收率，维持在安全区间。

| 需求编号 | 功能描述 | 用户故事 | 交互规格 |
|----------|----------|----------|----------|
| PM-WAT-01 | 回收率可视化 | 用户看到水回收系统的运作 | 水滴粒子动画，汇聚到回收槽 |

### 3.3 模块三：声音疗愈引擎

**模块目标**：这是项目的**核心心理干预手段**，通过重建自主神经节律来影响用户的心理状态。

**设计原则**：
- 声音是主要干预手段，视觉是辅助
- BPM ≈ 60-68，与静息心率同步
- 强烈避免情绪煽动
- 三层设计：低频环境层、中频呼吸层、高频觉醒层

#### 3.3.1 低频环境嗡鸣层

**功能描述**：模拟月球基地的机械环境音，提供"在场感"。

| 需求编号 | 功能描述 | 用户故事 | 技术规格 |
|----------|----------|----------|----------|
| SND-LOW-01 | 基地环境音 | 用户听到持续的、低沉的嗡鸣声 | 频率：40-80Hz，音量恒定 |
| SND-LOW-02 | 状态关联 | 环境音根据系统状态微调 | 氧循环异常时，嗡鸣出现轻微波动 |
| SND-LOW-03 | 空间化处理 | 用户感到声音来自环境 | 3D空间化，随视角变化 |

#### 3.3.2 中频呼吸引导层

**功能描述**：通过声音引导用户的呼吸节律。

| 需求编号 | 功能描述 | 用户故事 | 技术规格 |
|----------|----------|----------|----------|
| SND-MID-01 | 呼吸节拍 | 用户听到引导呼吸的节拍声 | BPM：60-68，每4拍为一次完整呼吸 |
| SND-MID-02 | 呼气/吸气标识 | 用户通过声音区分呼气吸气 | 呼气：长音渐弱，吸气：短音渐强 |
| SND-MID-03 | 灵活跟随 | 用户可以按自己节奏呼吸 | 引导是建议性的，非强制 |
| SND-MID-04 | 阶段适配 | 不同阶段有不同的引导密度 | 资源管理时弱，呼吸仪式时强 |

#### 3.3.3 高频觉醒提示层

**功能描述**：极少使用，用于标记重要的"觉醒点"。

| 需求编号 | 功能描述 | 用户故事 | 技术规格 |
|----------|----------|----------|----------|
| SND-HIGH-01 | 觉醒提示音 | 用户在重要时刻听到高频提示 | 频率：2000Hz以上，持续0.5秒 |
| SND-HIGH-02 | 使用限制 | 高频音极少出现 | 整个体验不超过3次 |
| SND-HIGH-03 | 触发时机 | 仅在单元开始/结束、仪式起止时使用 | 避免滥用导致脱敏 |

#### 3.3.4 整体音景编排规则

| 场景 | 主导层 | 辅层 | 备注 |
|------|--------|------|------|
| 资源管理时 | 低频嗡鸣 | 可选呼吸层 | 呼吸层可手动关闭 |
| 纪录/独白时 | 低频嗡鸣 | - | 降低其他声音 |
| 心理仪式时 | 呼吸引导层 | 低频嗡鸣（弱化） | 核心场景 |
| 地球升起时 | 低频嗡鸣 | - | 仅环境音 |
| 单元过渡时 | 高频提示 | 低频嗡鸣 | 标记切换 |

**模块三验收标准**：
- [ ] 用户反馈问卷中，"声音让我感到平静"得分≥4/5
- [ ] 用户反馈问卷中，"没有感到被声音打扰"得分≥4/5
- [ ] 音轨播放无卡顿、无循环痕迹

### 3.4 模块四：心理仪式系统（灵魂模块）

**模块目标**：这是项目的灵魂，是止观（禅修）的现代化表达。设计原则：不使用佛教术语，不谈修行，只谈"稳定"、"清醒"、"在此"。

**设计原则**：
- 每个"月球日"包含一个微型仪式
- Day 5 是完整的深度仪式（约7分钟）
- 仪式设计基于正念冥想的核心理论，但去宗教化

#### 3.4.1 仪式通用框架

**所有仪式共享的结构**：

| 阶段 | 时长 | 描述 | 用户动作 |
|------|------|------|----------|
| 入仪 | 30秒 | 提示即将开始仪式，关闭任务界面 | 点击"开始仪式" |
| 安身 | 60-90秒 | 呼吸引导+身体扫描语音 | 跟随语音，无需操作 |
| 定序 | 60-90秒 | 简单重复动作（如"按下这个阀门"） | 重复点击同一位置 |
| 观照 | 60-120秒 | 内在语音引导，开放式觉察 | 只需听，无需任何操作 |
| 出仪 | 30秒 | 仪式结束提示，回到基地日常 | 自动过渡 |

#### 3.4.2 七日仪式设计规格

| 日期 | 仪式名称 | 核心主题 | 特殊设计 |
|------|----------|----------|----------|
| Day 1 | 降落仪式 | 落地/安顿 | 首次仪式，侧重引导 |
| Day 2 | 空间仪式 | 熟悉空间 | 引导用户"标记"自己的角落 |
| Day 3 | 任务仪式 | 建立节奏 | 与资源管理任务结合 |
| Day 4 | 波动仪式 | 接受不确定 | 在系统异常后进行 |
| Day 5 | 深度仪式 | 内在探索 | 最完整版，7分钟时长 |
| Day 6 | 稳固仪式 | 强化习惯 | 简短版，侧重巩固 |
| Day 7 | 回望仪式 | 文明视角 | 从个人扩展到人类视角 |

#### 3.4.3 Day 5 深度仪式详细规格

**仪式名称**：此刻·月球

| 阶段 | 时长 | 内容描述 | 音频/视觉 |
|------|------|----------|-----------|
| 入仪 | 30秒 | "请放下手中的任务。我们花一点时间，在这里。" | 基地灯光渐暗至30% |
| 安身 | 90秒 | 呼吸引导："吸气...呼气...感受身体与座椅的接触..." | 呼吸层音节奏引导 |
| 定序 | 60秒 | "请反复按下这个阀门。每次按下，只是按下。" | 一个阀门光影出现，点击反馈为轻柔的"咔哒"声 |
| 观照 | 120秒 | 开放式引导："你可能会想到什么，或者什么都没有。无论是什么，只是知道。" | 音频引导，视觉极简 |
| 出仪 | 30秒 | "仪式结束。慢慢睁开眼睛。" | 基地灯光渐亮至100% |

**仪式引导语音示例（非宗教化表达）**：

> "现在，请把注意力放到呼吸上。不需要改变它，只是观察。
> 
> 吸气...呼气...
> 
> 如果你注意到自己在想什么，没有关系。那只是思绪。让它来，让它去。
> 
> 现在，把手放在这个阀门上。每次按下，只是按下。感受手指的动作，感受声音。
> 
> 不需要做得对，也不需要做得错。只是做。
> 
> 在这个空间里，在这个时间里，你只需要在这里。"

#### 3.4.4 交互设计规格

| 需求编号 | 功能描述 | 设计说明 |
|----------|----------|----------|
| RIT-INT-01 | 仪式入口 | 每个单元开始时，基地中央出现仪式光点，用户点击进入 |
| RIT-INT-02 | 动作反馈 | 重复动作有且仅有轻微的视觉+音频反馈，无任何奖励提示 |
| RIT-INT-03 | 跳过机制 | 用户可随时退出仪式回到基地，但记录"未完成" |
| RIT-INT-04 | 打断保护 | 仪式进行时，不被资源管理任务打断 |

**模块四验收标准**：
- [ ] 100%的测试用户完成Day 5深度仪式（可选择跳过，但目标0%跳过率）
- [ ] 用户反馈问卷中，"仪式后感到更平静"得分≥4/5
- [ ] 用户反馈问卷中，"仪式的语言表达让我感到舒适"得分≥4/5

```
│   │ 资源状态    │           │ 仪式入口    │        │
│   │（极简图标） │           │（光点）     │        │
│   └─────────────┘           └─────────────┘        │

│             └─────────────────────┘                │

```

### 6.3 关键交互流程

**流程一：开始体验**

```
1. 用户打开应用
   │
   ▼
2. 黑屏淡入（3秒）
   │
   ▼
3. 直接进入Day 1场景，无任何菜单
   │
   ▼
4. 播放Day 1开始独白
   │
   ▼
5. 用户开始体验
```

**流程二：进入仪式**

```
1. 仪式时间点到达
   │
   ▼
2. 基地中央出现柔和光点（约5cm直径）
   │
   ▼
3. 屏幕下方显示提示："仪式即将开始"
   │
   ▼
4. 用户点击光点
   │
   ▼
5. 场景渐暗，进入仪式流程
```

**流程三：资源管理交互**

```
1. 用户将鼠标悬停在资源对象上（太阳能板等）
   │
   ▼
2. 对象边缘出现柔和光晕（不改变形状）
   │
   ▼
3. 用户点击对象
   │
   ▼
4. 播放缓慢动画（2秒），同时播放低沉反馈音
   │
   ▼
5. 动画结束，用户可进行下一次操作或离开
```

## 七、技术约束与风险

### 7.1 技术约束

| 约束类型 | 约束内容 | 影响说明 |
|----------|----------|----------|
| 平台约束 | 首发PC（Windows 10+/macOS 10.15+） | VR版本需单独评估 |
| 引擎约束 | Unity 2022 LTS或Unreal Engine 5 | 切换引擎需重评估 |
| 音频约束 | 需支持立体声，空间化可选 | 环绕声非必须 |
| 存储约束 | 本地存储≤500MB用户数据 | 超过需优化 |
| 网络约束 | 体验过程无需网络（离线可用） | 确保断网可用 |

### 7.2 风险评估与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 节奏过慢导致用户无聊 | 中 | 高 | 保留30分钟时长，但确保每个单元有"锚点"（视觉/音频变化） |
| 过度安静导致不适感 | 中 | 中 | 环境嗡鸣层持续提供"在场感" |
| 被误认为冥想App | 高 | 中 | 产品命名、推广材料明确强调"文明模拟" |
| 技术实现超出预算 | 中 | 高 | MVP阶段砍掉所有非必要功能，专注四大模块核心体验 |

### 7.3 边界与排除项

**MVP明确排除项**：
- 用户账号系统
- 多人联机/社交功能
- 成就/徽章系统
- 外部API集成
- 云存储/同步
- 深度心理评估问卷
- 任何应用内购买/付费墙
