
这是一个非常清晰且具有深度的项目概念。将"文明级心理模拟"作为核心，区别于普通的娱乐游戏，意味着技术实现的优先级完全不同：**沉浸感、节制、反馈闭环和视听神经调节**是技术重点，而非复杂的游戏机制或联网功能。

基于你提供的创意框架，以下是针对 Unity 开发环境的**系统需求规格说明书（SRS）与功能规格（FRS）草案**。

---

# 《Lunar Week One》系统需求与功能规格说明书 (MVP阶段)

## 1. 系统架构概览

### 1.1 技术栈
*   **核心引擎**：Unity 2022 LTS (推荐使用 URP 以优化移动端/VR 性能，或 HDRP 追求极致 PC 视觉)
*   **平台**：PC (Windows/Mac) 为 MVP 首选；VR (SteamVR/Oculus) 作为扩展选项
*   **开发语言**：C#
*   **构建工具**：Unity Timeline (负责时间轴叙事), Cinemachine (负责镜头律动), Post-Processing (负责视觉氛围)

### 1.2 核心设计原则
*   **零 HUD 设计**：所有信息必须环境化，即显示在游戏世界内的物体上（如屏幕、仪表盘），而非 2D UI 覆盖层。
*   **音频驱动体验**：音频系统拥有最高优先级，直接控制视觉参数（如呼吸节奏影响光照强弱）。
*   **极简交互**：交互仅限于“注视 + 点击”或“长按”，避免复杂操作破坏心流。

## 2. 功能模块详细规格

### 模块一：模拟建筑与资源管理系统
**目标**：建立秩序感，提供低焦虑的控制感。

| 功能点        | 详细需求                | 交互逻辑               | 反馈机制                                             |
| :--------- | :------------------ | :----------------- | :----------------------------------------------- |
| **太阳能板维护** | 玩家需定期调整太阳能板角度以维持能源。 | 拖拽滑块或在 3D 空间中抓取旋转。 | 角度正确时：灯光变柔和，背景音频率降低。<br>角度错误时：灯光闪烁（红色警告，但不导致死亡）。 |
| **维生系统监控** | 氧气与水循环的抽象可视化。       | 纯观察性，偶尔点击"重启过滤阀"。  | 声音层增加"流动感"。仪表盘指针在安全区间内摆动。                        |
| **环境噪音控制** | 基地外围的风暴或陨石撞击声。      | 不可控，模拟环境不确定性。      | 音频随事件波动；窗户玻璃微震（摄像机震动）。<br>**关键点**：内部维持静谧，对比外部狂暴。 |


**技术实现**：
*   使用 `ScriptableObject` 定义资源数据（能源、氧气、水）。
*   观察者模式：当资源数值变化时，驱动 Material 的 Emission 强度和 Audio Mixer 的 Volume 参数。

---

### 模块二：导演化叙事系统
**目标**：存在感驱动，非线性叙事碎片。

| 功能点           | 详细需求                  | 触发条件               | 表现形式                                      |
| :------------ | :-------------------- | :----------------- | :---------------------------------------- |
| **历史回响 (纪录)** | 播放真实的阿波罗计划或未来探索的音频片段。 | 进入特定房间或时间到达（如第3天）。 | 全息投影或 CRT 屏幕播放。音频使用空间化音效，像是从隔壁传来。         |
| **内心独白 (剧情)** | 第一人称文字/语音，描述宇航员的感受。   | 资源维护完成后，或长时间静坐。    | UI 环境化（显示在手腕终端或玻璃上）。文字淡入淡出极慢。             |
| **地球升起 (空镜)** | 视觉奇观，提供文明视角。          | 特定的"休息日"或仪式结束后。    | 摄像机自动推向舷窗，Post-processing 开启 Bloom（光晕）效果。 |

**技术实现**：
*   Unity Timeline 掌控整个 30 分钟的总流程。
*   使用 `Signal Emitter` 触发 AudioSource 播放和 UI 文本显示。
*   玩家位置检测：使用 `Vector3.Distance` 或 Trigger 判断玩家是否处于叙事触发区。

---

### 模块三：音频疗法与神经调节系统
**目标**：**最核心模块**。自主神经节律重建。

| 功能点        | 详细需求                             | 技术指标                                     |
| :--------- | :------------------------------- | :--------------------------------------- |
| **基础环境底噪** | 模拟空间站/基地的低频嗡鸣。                   | 持续播放，使用 Loop。50Hz-80Hz 低频强调。             |
| **呼吸同步引导** | 通过声音节奏引导用户呼吸（4秒吸，4秒呼 = 7.5 BPM）。 | **双耳节拍**：左右耳微弱频率差。音量起伏与呼吸同频。             |
| **动态混音**   | 根据玩家心率（模拟值）或交互紧张度调整混响。           | 紧张时：减少高频，增加低频。<br>平静时：引入微弱的高频"风铃"声作为正反馈。 |

**技术实现**：
*   **Audio Mixer**: 设置多个 Snapshot（Normal, DeepMeditation, Warning）。根据游戏状态平滑过渡（`AudioMixer.TransitionTo`）。
*   **节奏控制**: 编写一个 `BreathGuide` 脚本，通过正弦波 `Mathf.Sin(Time.time)` 控制音频源的 Pitch 或 Volume，形成自然的"潮汐感"。

---

### 模块四：心理仪式系统
**目标**：止观的现代化表达，心智稳定核心。

| 功能点 | 详细需求 | 交互逻辑 |
| :--- | :--- | :--- |
| **入仪/出仪** | 屏幕全黑，仅保留呼吸引导音。 | 点击任意键开始。结束时机由系统判断（如坚持 2 分钟）。 |
| **身体扫描** | 引导音提示关注身体部位（脚、腿、 torso...）。 | 结合视觉效果：身体部位对应的 UI 区域微弱亮起（VR 版中可高亮玩家身体）。 |
| **定序动作 (阀门)** | 重复开启/关闭氧气阀门的行为，作为冥想的"念珠"。 | 长按 4 秒吸气（阀门打开动画），松开 4 秒呼气（阀门关闭）。 |
| **悬浮观察** | 自由观察漂浮的尘埃颗粒或月球表面。 | 鼠标/手柄极其缓慢地移动视角，模拟失重凝视。 |

**技术实现**：
*   **时间膨胀**：在仪式进行时，修改 `Time.timeScale`（如减慢至 0.95）或调整动画播放速度，营造"时空停滞"感。
*   **粒子系统**：使用 Unity Particle System 制作在光束中漂浮的尘埃粒子，作为视觉锚点。

## 3. 用户体验流程 (Journey Map & State Machine)

系统需维护一个全局状态机来管理 7 个 "月球日"。

```csharp
public enum LunarDayState {
    Day1_Arrival,       // 抵达：混乱、警报
    Day2_Adaptation,    // 适应：熟悉界面、资源稳定
    Day3_Order,         // 秩序：任务执行、建立规律
    Day4_Uncertainty,   // 不确定：模拟系统故障（视觉干扰）
    Day5_Ritual,        // 仪式：核心冥想体验
    Day6_Stability,     // 稳定：自动化运行、平静
    Day7_Reflection     // 回看：地球升起、总结
}
```

*   **Day 4 设计细节**：在"建立秩序"后，通过制造视觉故障（如 Glitch Effect 或 屏幕噪点）来打破稳定，引入焦虑，逼迫玩家进入 Day 5 的内在调节。
*   **Day 5 设计细节**：剥夺玩家的控制能力（所有操作面板失效），只留"呼吸阀"可用，强制切断对外的控制欲，转向内在。

## 4. MVP 技术规格与限制

### 4.1 性能指标
*   **帧率**：PC 端稳定 60 FPS 以上（VR 端需 72/90 FPS）。
*   **加载时间**：场景切换加载不超过 3 秒（采用流式加载或单场景多区域）。
*   **内存**：MVP 总资源包体控制在 2GB 以内（音频文件需压缩，使用 Vorbis/PCM 根据平台选择）。

### 4.2 数据流与存储
*   **无后端**：MVP 阶段完全本地化，不涉及云端。
*   **本地存储 (`PlayerPrefs`)**：
    *   记录当前进度（第几天）。
    *   记录上次体验结束的时间点。
    *   *可选*：记录玩家完成一次"呼吸仪式"的平均时长（用于分析）。

### 4.3 视觉风格
*   **PBR 材质**：冷色调金属、磨砂玻璃、粗糙混凝土。避免高饱和度颜色。
*   **光照**：强调体积光，光源色温偏冷（月球表面日光）或暖（内部人造光）。
*   **后期处理**：
    *   **Color Grading**: 降低对比度，整体偏灰蓝，营造"静谧"。
    *   **Depth of Field**: 引导玩家关注近处物体或特定焦点。
    *   **Film Grain**: 增加少量胶片颗粒，增加真实感和质感。

## 5. 开发路线图 建议

**阶段一：灰盒与核心循环 (Week 1-2)**
*   搭建月球基地内部灰盒。
*   实现玩家移动与简单的交互（点击、长按）。
*   实现"呼吸阀"的最基本逻辑（按住/松开）。

**阶段二：音频系统与氛围 (Week 3-4)**
*   **关键里程碑**：集成音频混合器，实现呼吸音效逻辑。
*   导入临时测试音效，调整 3D 空间音效参数。
*   添加基本的后期处理效果。

**阶段三：叙事与 7 日流程 (Week 5-6)**
*   使用 Unity Timeline 编排 7 天的事件。
*   完成 Day 1 (混乱) -> Day 3 (稳定) -> Day 5 (仪式) 的流程跑通。
*   实现 Day 4 的视觉故障效果。

**阶段四：打磨与测试 (Week 7-8)**
*   替换为高质量美术资产与音效。
*   **内部测试**：验证 30 分钟内玩家是否感到焦虑、无聊，还是"安静而清醒"。
*   调整呼吸引导的节奏（4-4-4-4 还是 4-7-8？推荐 4-6 以保持清醒）。
